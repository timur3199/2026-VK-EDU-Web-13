Как правильно использовать prefetch_related и select_related в Django ORM?

Столкнулся с классической проблемой N+1 запросов при работе с Django ORM. Пробовал использовать select_related, но он не работает для полей ManyToManyField.

Вот мой код:

questions = Question.objects.all()
for q in questions:
    print(q.author.username)   # N+1 здесь
    print(q.tags.all())         # и здесь

Как правильно оптимизировать этот код? Когда использовать prefetch_related, а когда select_related? Есть ли разница в количестве SQL-запросов?

Краткое правило:

  • select_related — для ForeignKey и OneToOne (JOIN в одном SQL-запросе)
  • prefetch_related — для ManyToMany и обратных ForeignKey (отдельный SQL + Python join)

Оптимизированный код:

questions = Question.objects.select_related('author').prefetch_related('tags')

Теперь Django сделает 2 запроса вместо 2N+1. select_related подтянет авторов через JOIN, а prefetch_related — теги отдельным запросом с WHERE id IN (...).

Добавлю про Prefetch объект — он даёт больше контроля. Например, можно отфильтровать prefetch или задать order:

from django.db.models import Prefetch

questions = Question.objects.prefetch_related(
    Prefetch('answers', queryset=Answer.objects.order_by('-created_at'))
)

Используйте django-debug-toolbar чтобы видеть реальное количество запросов.

Ещё один момент: select_related тоже можно использовать с глубокой вложенностью через двойное подчёркивание:

Question.objects.select_related('author__profile')

Будет JOIN сразу трёх таблиц в одном запросе.
